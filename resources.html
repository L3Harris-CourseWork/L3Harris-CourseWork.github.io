Technical Resources

Unit Testing

Software development projects involve testing at various stages, with software developers taking responsibility for unit testing at the class level. When code is written within a class, developers are tasked with creating unit tests to validate code functionality for both normal and error scenarios. These tests should be executed using a testing framework like JUnit, which facilitates test execution and result verification. To ensure thorough testing, each class should be tested in isolation, necessitating the control of all dependencies the tested class relies on with other objects. Object mocking is a technique used to create objects that the tested class utilizes during testing. Mocking frameworks, such as Mockito, aid in defining mock objects and their expected behaviors when methods are called. Additionally, these frameworks enable tests to verify if methods were called the expected number of times during testing. Your team will be expected to develop unit tests for all non-trivial classes within your project.

The Unit testing technology that you select will obviously be highly dependent on the technology that you choose to implement your project with. However, be sure that the unit testing technology will properly integrate with GitHub and the hosting platform Render. You should additionally feel free to search for other possible unit testing technologies as you desire as there are many that are not encapsulated in this list.

  • JUnit
  • JMockit
  • Typemock
  • Karma
  • Jasmine
  • Mocha
  • Ruby Unit Testing

GitHub Tools

Your team will be using GitHub as your version control repository. There are a wide range of techniques and tools for working with GitHub. There are tools with robust GUIs, or you may choose to use a simple command line interface. While the tool you use to integrate with GitHub is your decision, it is VERY HIGHLY recommended that you DO NOT only interact with GitHub through their web interface. This will prove to be cumbersome and fraught with challenges.

  • GitHub Desktop - GitHub Desktop is an official GitHub client that provides a graphical interface for managing your repositories. It's user-friendly and suitable for both beginners and experienced developers.
  • GitKraken - GitKraken is a cross-platform Git client that supports GitHub. It has a visually appealing interface and offers features like a drag-and-drop Git branching system.
  • SourceTree - A free Git client that works with GitHub repositories. It provides a visual representation of branches and their commits.
  • GitExtensions - A Git client for Windows that integrates with GitHub. It provides a GUI for Git, including features like visualizing the commit history.
  • VSCode - A popular code editor that has built-in Git integration, making it easy to work with GitHub repositories directly from the editor.
  • GitUp - A Git interface for macOS that focuses on simplicity and a visual representation of the Git history.
  • TortoiseGit - A Windows Shell Interface to Git. It integrates with the file explorer, allowing you to perform Git operations through context menus.
  • Tower - A Git client for Mac and Windows that provides a powerful interface for interacting with Git repositories, including GitHub.
