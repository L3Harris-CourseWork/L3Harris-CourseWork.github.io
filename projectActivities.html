Github/Github Issues

You will conduct a short hello world activity using Github. Understanding Git/Github is not only important for this project, but is important for understanding the benefits and usage of version control systems throughout the software engineering process. Additionally, Github issues is similarly a beneficial tool to know regarding bug tracking.

Activity

You will conduct several activities for this task:
  • You should create an individual Github repository and use this tutorial. Note, that there are many high-quality Github tutorials available that you should feel free to reference.
  • Your team should create a single Github repository. One team member should create a simple text file containing their name. Each team member should "pull" this file, add their name and submit it to the central repository. Ultimately, each team member should have a checkin and the file should contain everyone's names. Additionally, to better understand the activity, your team is encouraged to create and mitigate merge conflicts.
  • Complete the tutorial for Github issues. Create several known and/or test defects in your application.

Deliverable

Provide the link to your team’s GitHub repository to the designated submission area in Mycourses. Note, that you should submit this template (you can leave the aspects of the template that have not yet been covered blank for now).

Unit Testing

Unit testing is a software testing technique where individual units or components of a software application are tested in isolation to ensure they function as intended. A "unit" in this context is the smallest testable part of the software, typically a single function, method, or procedure. The primary goal of unit testing is to validate that each unit of the software performs as designed, helping to identify and fix bugs early in the development process. You will utilize it for your project. The specific form of unit testing that you use will be based upon your selected language.

Activity

  • Identify a proper unit testing technology for your selected language. For example, you can use this tutorial for Ruby Unit Testing.
  • Create five “hello world” unit tests and integrate them into your platform.

Deliverable

You should also submit a screenshot of the passing unit tests provided from Render. Note, that you should submit this template. Each component of the template should now be complete.
